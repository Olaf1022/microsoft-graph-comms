Automated Testing

The testing file explained how to use ngrok to setup your pipeline. This file explains how to automate the testing using the Microsoft.Graph.Testing.Localhost nuget.

The library

The Microsoft.Graph.Testing.Localhost library acts as an automation layer to run and manage the tunnels dynamically to your localhost. It provides support for ngrok out of the box.

Note

The current version of the library does not support end to end encrypted tunnels.

The IProxyServiceProvider wraps the implementation of different tunneling options to provide an unified mechanism to manage the tunnels.

Ngrok

  • Follow the steps in testing to get an authentication token for ngrok.

  • Create an instance of NgrokProxyServiceProvider by passing in the auth token.

var ngrokProxy = new NgrokProxyServiceProvider(authToken);
  • For the purposes of the bot, we require two tunnels. One for media and one for signaling. Add the following lines to setup the tunnels.
await ngrokProxy.RegisterServiceAsync("signaling", ProxyServiceMode.Http, localSignalingPort).ConfigureAwait(false);
await ngrokProxy.RegisterServiceAsync("media", ProxyServiceMode.Tcp, localMediaPort).ConfigureAwait(false);
  • Call the startup server method to finish setting up the tunnels
await NgrokServiceProvider.StartProxyAsync().ConfigureAwait(false);
  • Retrieve the tunnel details for passing in to the SDK.
var serviceDetails = await ngrokProxy.GetServiceAsync("signaling").ConfigureAwait(false);
var mediaServiceDetails = await ngrokProxy.GetServiceAsync("media").ConfigureAwait(false);
  • The different service details objects have the values required to be passed in as mentioned in testing.
// Setup the signaling
var remoteUri = new UriBuilder("https", serviceDetails.RemoteAddress).Uri;
builder.SetServiceBaseUrl(platformUri);

// Setup media
var ngrokTcpServerNumber = mediaServiceDetails.RemoteAddress.Split('.')[0];
var mediaServiceFqdn = $"{ngrokTcpServerNumber}.bot.contoso.com";

// In media
var mediaPlatform = new MediaPlatformSettings 
{
    ...
    MediaPlatformInstanceSettings = new MediaPlatformInstanceSettings
    {
        ...
        InstanceInternalPort = localMediaPort,
        InstancePublicPort = mediaServiceDetails.RemotePort,
        ...
    },
}
